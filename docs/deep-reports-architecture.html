Orchestrator engine

Same pipeline, three different runtimes — pick per deployment

NativeOrchestrator

in-memory · zero dependencies
  • PipelineState enum: DRAFTED → CHECKED → SCORED → REFINED → DONE
  • Routing is a plain if/elif chain reading agent output fields directly
  • Refine loop is a while iteration <= 3 Python loop
No checkpointing — a crash mid-run loses pipeline state. Fastest to start, hardest to resume.

LangGraphOrchestrator

StateGraph · conditional edges
  • Each agent is a graph node; TypedDict state passed by reference between nodes
  • add_conditional_edges() encodes the ≥3 blockers / score <5 branches
  • Optional checkpointer persists state — resumable after failure
More setup, but you get replay, time-travel debugging, and durable execution for free.

CrewAIOrchestrator

Agent + Task · Crew
  • Four Agent objects with role/goal/backstory; one Task per agent
  • Process.sequential — refine loop is simulated by re-appending tasks
  • Delegation and tool-use come from CrewAI's own agent framework
Native conditional routing is limited — the ≥3 blockers branch is implemented as a custom callback, not a first-class edge.
